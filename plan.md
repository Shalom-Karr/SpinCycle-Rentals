# SpinCycle Rentals: Customer Portal & PWA Expansion Plan

## Overview
This document outlines the architecture and implementation steps for expanding the SpinCycle Rentals static website into a fully functional web application. The goal is to allow customers to log in, view their rental status, download invoices, and install the website as a native app on their phones or desktops—all while maintaining our blazing-fast, "zero-build" Vanilla JS architecture.

## Tech Stack
*   **Frontend:** HTML5, Vanilla CSS, Vanilla JS (No React/Webpack).
*   **Backend & Auth:** Supabase (PostgreSQL, GoTrue Auth, Row Level Security).
*   **Payments & Invoicing:** Stripe API & Stripe Customer Portal.
*   **App Conversion:** Progressive Web App (PWA) standards (Service Workers, Web App Manifest).

---

## Phase 1: Authentication & Database (Supabase)
To give users a secure portal, we need a backend to manage their identities and rental data.
*   [ ] **Initialize Supabase Project:** Set up a new Supabase project for SpinCycle Rentals.
*   [ ] **Configure Supabase Auth:** Enable Magic Link (Passwordless) or Google OAuth to reduce login friction for customers.
*   [ ] **Database Schema:** Create a `customers` table linked to `auth.users` to store rental status, machine serial numbers, and Stripe Customer IDs.
*   [ ] **Row Level Security (RLS):** Implement strict RLS policies so users can only `SELECT` their own profile and invoice history.

## Phase 2: The Customer Dashboard
Build the secure, client-side rendered dashboard using Vanilla JS.
*   [ ] **Login UI (`login.html`):** Create a clean login page that interfaces with the Supabase JS client.
*   [ ] **Dashboard UI (`portal.html`):** Build the authenticated layout. If a user is not logged in, redirect them back to `login.html`.
*   [ ] **Stripe Integration:** Use Supabase Edge Functions (or Zapier/Make) to listen to Stripe Webhooks. When a monthly $45 payment succeeds, log the invoice in the Supabase database.
*   [ ] **Invoice History:** Use Vanilla JS to fetch the user's invoice history from Supabase and render it in a clean HTML table, providing links to download PDF receipts (generated by Stripe).
*   [ ] **Billing Management:** Add a secure button that redirects the user to their Stripe Customer Portal session so they can easily update their credit card on file or view past billing history directly through Stripe.

## Phase 3: Progressive Web App (PWA) Conversion
Turn the static website into an installable application for iOS, Android, and Desktop.
*   [ ] **Web App Manifest (`manifest.json`):** Create a JSON file defining the app's name ("SpinCycle"), theme colors, display mode (`standalone`), and app icons (192x192, 512x512).
*   [ ] **Service Worker (`sw.js`):** Write a Vanilla JS service worker to cache core assets (`index.html`, `style.css`, `app.js`, logos). This ensures the app loads instantly, even on poor cellular connections, and works flawlessly when launched from a phone's home screen.
*   [ ] **Install Prompt:** Add a custom "Install App" button in the navigation or portal that triggers the native `beforeinstallprompt` event, allowing users to download the app directly to their home screen without needing to go through the iOS App Store or Google Play Store.

## Phase 4: Polish & Deployment
*   [ ] **Secure Environment Variables:** Ensure Supabase anon keys and Stripe public keys are safely injected into the static files.
*   [ ] **Testing:** Verify the login flow, database RLS security, and PWA installation prompts across Chrome, Safari, iOS, and Android.
*   [ ] **Go Live:** Push changes to the `main` branch and let the static hosting platform (like Cloudflare Pages or GitHub Pages) automatically deploy the new PWA portal.
